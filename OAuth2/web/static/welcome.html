Client sends user to google with a client id and client secret, along with a 'state' code that randomly generates.
After the user allows permissions, google then sends back a code for use for the client. The client then verifies that code at googles 'token' website.
Google then sends back a token for use by the client for the user.
The client then gets the user's first name, last name, and email and displays them to the user along with the original 'state' variable used when the user was first sent to google.
